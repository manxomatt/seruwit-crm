Titik Integrasi
GrnConfirmationService → StockMovementRecorder::record() — sudah ada, langsung dipakai. Source type: 'grn', source_id: grn_item.id
PO partner_id → FK ke tabel partners (Partners module) — supplier difilter berdasarkan tipe 'supplier' atau 'both'
PO item product_id → FK ke tabel products (Product module) — batch_number + expiry_date disimpan di GRN item untuk FMCG
GRN location_id → FK ke warehouse_locations — per item bisa masuk lokasi/bin berbeda dalam satu GRN
LowStockNotifier → Dipanggil setelah GRN confirmed untuk recalculate stok dan clear low-stock alert jika sudah di atas threshold
